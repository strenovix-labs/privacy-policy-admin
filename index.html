1. About VoteAble

VoteAble is a secure election management and voter assistance platform designed for authorized election officials including:

  • Booth Level Officers (BLO)
  • Returning Officers (RO)
  • Assistant Returning Officers (ARO)
  • Super Admin

The system enables field-level coordination, voter assistance tracking, and geospatial visualization of voter needs, especially for Differently Abled Persons (DAP).

System Owner: Election Administration (Internal Use System)
Developer Contact: strenovix@gmail.com

2. Data Collected

Identity & Administrative Data

  • Name (Voters and Officers)
  • Mobile Number
  • Voter ID (EPIC Number)
  • Gender, Age and Date of Birth

Location & Mapping Data

  • Voter location (Latitude & Longitude)
  • Officer real-time location (when app in active use)
  • Address data (Block, Taluk, District, Pincode)

Assistance & Sensitive Data

  • Disability type
  • Assistance requirements
  • Voting intent (Will Vote / Won’t Vote)
  • Assistance status and resolution tracking

Operational Metadata

  • Timestamps of updates, Query resolution status, Officer assignments

3. Purpose of Processing

  • Mapping voters and identifying assistance requirements
  • Displaying Differently Abled Persons (DAP) on secure map interfaces
  • Enabling officers to locate and assist voters efficiently
  • Managing voter queries and tracking resolution status
  • Supporting decision-making via dashboards and analytics
  • Coordinating field-level election activities
No Commercial UseNo AdvertisingNo Profiling

4. Location Transparency

Important — Location Usage

  • Officer location collected only during active app usage
  • Used to show nearby voters requiring assistance
  • Enables real-time operational coordination
  • Not used for background tracking

Voter locations are stored as part of official election records and used for mapping and assistance planning.

7. Permissions Used

ACCESS_FINE_LOCATIONShow precise location on map
ACCESS_COARSE_LOCATIONApproximate fallback location

8. Storage & Retention

Data stored securely in PostgreSQL database hosted via Railway infrastructure. Retention aligned with election operational requirements. Officer location is not permanently stored, only used during active sessions.

Data modification governed by administrative privileges.

10. Security Measures
